# Great-Procreations

**Role in the ecosystem:** Browser-based deep ancestry visualization. Renders pedigree charts of arbitrary depth using D3.js and an original radial layout algorithm, deployed as a GitHub Pages site.

## Intent & Invariants
- **Intent:** Provide a high-density, interactive "sunburst of ancestry" that remains legible beyond the 5th generation.
- **Invariant:** The center of the radial graph is always the "subject" (Generation 0).
- **Invariant:** Each concentric ring MUST represent exactly one ancestral generation.
- **Invariant:** Layout is strictly radial; do not "helpfully" refactor to rectilinear or force-directed layouts as it breaks the comparative generation view.

## Key Data Contracts
- **`data/core-pedigree.csv`**: The lookup table linking `ascendancy` IDs to FamilySearch IDs (`fid`).
- **`ascendancy` ID (Ahnentafel Numbering)**:
    - `1`: Subject
    - `2n`: Father of `n`
    - `2n + 1`: Mother of `n`
- **Prefix Convention**: Data slices are prefixed with `slice-` and padded to 3 digits (e.g., `data/people/slice-031.csv`).

## Execution Model
- **Environment:** Vanilla HTML/D3.js (v3) hosted on GitHub Pages.
- **State Management:** The current root person is managed via the URL hash (e.g., `#1024`).
- **Data Loading:** Uses `d3.csv` for lazy loading of data "slices." Do not transition to a monolithic JSON bundle; the dataset grows too large for single-fetch browser environments.

## Ecosystem Boundaries
- **Upstream:** Consumes CSV exports generated by [jacobmgreer/Family-Tree](https://github.com/jacobmgreer/Family-Tree).
